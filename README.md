# Domain Analyzer

Мощный инструмент для автоматического сбора и анализа данных о доменах. Парсит содержимое сайтов, извлекает контакты и сохраняет результаты в SQLite для дальнейшего анализа.

## Возможности

- **Парсинг контента**: Извлечение title, description, h1 и текстового содержимого главной страницы
- **Контактная информация**: Автоматическое обнаружение телефонов, email-адресов, ИНН организаций
- **Социальные сети**: Определение ссылок на социальные сети (VK, Telegram, Facebook, Instagram, Twitter, YouTube)
- **Адреса**: Извлечение почтовых адресов из текста страницы
- **Многопоточность**: Настраемое количество параллельных запросов
- **Повторные попытки**: Автоматический переобход при ошибках
- **Веб-интерфейс**: Удобная админ-панель для просмотра результатов
- **Экспорт в CSV**: Выгрузка всех данных в формате CSV
- **Логирование**: Полный лог всех действий с timestamp

## Установка

### Требования

- Go 1.22+
- SQLite3

### Сборка

```bash
go build -o domain-analyzer ./cmd/main.go
```

## Использование

### Базовый запуск

```bash
./domain-analyzer -domains domains.txt
```

### С очисткой БД и логов перед запуском

```bash
./domain-analyzer -domains domains.txt -clean
```

### С указанием параметров

```bash
./domain-analyzer -domains domains.txt -workers 10 -delay 500 -retry 3 -db data.db -log-dir logs
```

## Параметры CLI

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `-domains` | Путь к файлу с доменами (обязательно) | - |
| `-workers` | Количество параллельных потоков | 5 |
| `-delay` | Задержка между запросами (мс) | 1000 |
| `-retry` | Количество попыток при ошибке | 3 |
| `-db` | Путь к файлу SQLite | domains.db |
| `-log-dir` | Директория для логов | logs |
| `-clean` | Очистить БД и логи перед запуском | false |

## Формат файла доменов

Каждый домен на новой строке. Пустые строки и строки, начинающиеся с `#`, игнорируются.

```
example.com
google.com
# этот домен будет игнорирован
yandex.ru
```

## Веб-интерфейс

После запуска веб-интерфейс доступен по адресу: **http://localhost:1439**

### Функции веб-интерфейса

- **Статистика**: Общее количество доменов, успешных и ошибок
- **Время работы**: Время запуска, время работы, расчётное время завершения
- **Консоль логов**: Real-time логирование процесса парсинга
- **Таблица доменов**: Просмотр всех собранных данных с пагинацией (1000 на страницу)
- **Поиск**: Поиск по любому полю (домен, title, телефоны, email, адрес, ИНН, соцсети, статус)
- **Фильтр статуса**: Фильтрация по успешным или неуспешным доменам
- **Экспорт**: Выгрузка данных в CSV

## База данных

### Таблица `domains`

| Колонка | Тип | Описание |
|---------|-----|----------|
| id | INTEGER | ID записи |
| domain | TEXT | Домен |
| url | TEXT | Полный URL |
| title | TEXT | Title страницы |
| description | TEXT | Meta description |
| h1 | TEXT | Заголовок H1 |
| text_content | TEXT | Текстовое содержимое |
| phones | TEXT | Телефоны (через запятую) |
| emails | TEXT | Email (через запятую) |
| inn | TEXT | ИНН организации |
| social_media | TEXT | Ссылки на соцсети |
| address | TEXT | Адрес |
| status | TEXT | Статус (success/failed) |
| error_message | TEXT | Сообщение об ошибке |
| scraped_at | TEXT | Timestamp парсинга |

## API

Веб-интерфейс предоставляет REST API:

### GET /api/domains

Параметры:
- `page` — номер страницы (по умолчанию: 1)
- `limit` — количество на странице (по умолчанию: 1000)
- `search` — поисковый запрос
- `searchField` — поле для поиска
- `status` — фильтр по статусу (success/failed)

### GET /api/stats

Возвращает статистику: количество успешных и неуспешных доменов.

### GET /api/startinfo

Возвращает информацию о запуске: время старта, общее количество доменов, обработано.

### GET /api/logs

Server-Sent Events поток логов.
